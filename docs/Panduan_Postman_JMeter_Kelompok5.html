BAGIAN A — API TESTING DENGAN POSTMAN

A1. Persiapan

Langkah 1: Import Collection

  1. Buka aplikasi Postman
  2. Klik Import (pojok kiri atas)
  3. Pilih file docs/WebSekolah_Postman_Collection.json
  4. Klik Import — collection "Web Sekolah Terintegrasi - API Testing" muncul di sidebar

Langkah 2: Buat Environment

  1. Klik ikon Environment (gear icon kanan atas)
  2. Klik Add → beri nama: Web Sekolah Production
  3. Tambah variable:
    • Variable: base_url
    • Initial Value: https://web-sekolah.duckdns.org
    • Current Value: https://web-sekolah.duckdns.org
  4. Klik Save
  5. Pilih environment "Web Sekolah Production" dari dropdown kanan atas
[Screenshot: tampilan environment dengan base_url terisi]

Langkah 3: Aktifkan Cookie Jar

  1. Di Postman, buka Settings → General
  2. Pastikan "Automatically follow redirects" = ON
  3. Pastikan "Send cookies" = ON (agar session cookie tersimpan setelah login)

A2. Urutan Pengujian

Jalankan request DALAM URUTAN BERIKUT. Login harus dilakukan dulu sebelum request lain karena aplikasi menggunakan session-based auth.

Skenario 1: Login sebagai Admin → Test semua endpoint admin

  1. Jalankan 1.1 Login (Admin) → cek status 200/302 → PASS
  2. Jalankan 2.1 Search Users - Keyword valid
  3. Jalankan 2.2 Search Users - Keyword < 2 char
  4. Jalankan 2.3 Search Users - Keyword tidak ada hasil
  5. Jalankan 3.1 Search Siswa - Keyword nama
  6. Jalankan 3.2 Search Siswa - Keyword NIS
  7. Jalankan 4.1 Search Jurnal - Keyword materi
  8. Jalankan 7.1 GET Dashboard s/d 7.6 GET Profil

Skenario 2: Login sebagai Guru BK → Test endpoint BK

  1. Jalankan 1.2 Login (Guru BK)
  2. Jalankan 5.1 Search BK - Keyword nama siswa
  3. Jalankan 7.3 GET BK

Skenario 3: Login sebagai Guru → Test RBAC

  1. Jalankan request login manual dengan username: guru1, password: password123
  2. Jalankan 7.8 GET /users (RBAC test) → harus redirect (302) atau HTML dashboard

Skenario 4: Login sebagai Siswa → Test dashboard siswa

  1. Jalankan 1.3 Login (Siswa)
  2. Jalankan 7.7 GET Siswa Dashboard

BAGIAN B — PERFORMANCE TESTING DENGAN APACHE JMETER

B1. Persiapan JMeter

  1. Download Apache JMeter dari https://jmeter.apache.org/download_jmeter.cgi (versi 5.6+)
  2. Extract ZIP → jalankan bin/jmeter.bat (Windows)
  3. JMeter GUI terbuka

B2. Load Testing — Skenario Normal

Tujuan: Mengukur performa halaman /dashboard dan /bk saat diakses oleh beberapa pengguna bersamaan dalam kondisi normal.

Setup Thread Group (Load Test)

  1. Klik kanan pada Test Plan → Add → Threads → Thread Group
  2. Set parameter:
    • Number of Threads (users): 50
    • Ramp-up Period: 10 detik
    • Loop Count: 3
  3. Klik kanan Thread Group → Add → Sampler → HTTP Request
  4. Isi HTTP Request:
    • Server Name: web-sekolah.duckdns.org
    • Protocol: https
    • Path: /dashboard
    • Method: GET
  5. Tambah HTTP Request kedua untuk path /bk
  6. Klik kanan Thread Group → Add → Listener → Summary Report
  7. Klik kanan Thread Group → Add → Listener → View Results Tree
[Screenshot: JMeter Thread Group dengan 50 users, ramp-up 10 detik, loop 3]

Jalankan Load Test

  1. Klik tombol Run (segitiga hijau) di toolbar
  2. Tunggu hingga selesai (~30 detik)
  3. Klik Summary Report untuk melihat hasil
  4. Catat nilai: # Samples, Average, Min, Max, Error%, Throughput

B3. Stress Testing — Temukan Batas Server

Tujuan: Menemukan titik di mana server mulai menunjukkan error (Error% naik signifikan).

Skenario Stress Test Bertahap

Ulangi test dengan jumlah user yang meningkat, catat hasilnya:

  1. Tahap 1: 100 users, ramp-up 10s, loop 2 → catat Error%
  2. Tahap 2: 200 users, ramp-up 10s, loop 2 → catat Error%
  3. Tahap 3: 500 users, ramp-up 10s, loop 2 → catat Error%
  4. Jika Error% masih <5%, naikkan lagi sampai Error% meningkat signifikan (>10%)
  5. Catat jumlah user di mana Error% mulai >10% — itu batas kapasitas server

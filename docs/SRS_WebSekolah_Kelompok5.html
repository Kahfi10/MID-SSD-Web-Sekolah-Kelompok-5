1. Pendahuluan

1.1 Tujuan Dokumen

Dokumen Software Requirements Specification (SRS) ini mendeskripsikan kebutuhan fungsional dan non-fungsional dari sistem Web Sekolah Terintegrasi. Dokumen ini ditujukan kepada tim pengembang sebagai panduan implementasi, tim penguji sebagai acuan pengujian, dan dosen pengampu sebagai evaluasi capaian proyek.

Dokumen ini mengacu pada standar IEEE 830-1998 untuk penulisan Software Requirements Specification.

1.2 Ruang Lingkup Sistem

Nama Sistem: Web Sekolah Terintegrasi

URL Produksi: https://web-sekolah.duckdns.org

Repository: https://github.com/Kahfi10/MID-SSD-Web-Sekolah-Kelompok-5

Sistem ini adalah aplikasi manajemen sekolah berbasis web yang mengintegrasikan 5 modul utama: Manajemen Pengguna, Data Kesiswaan, Jurnal Mengajar, Bimbingan Konseling (BK), dan Absensi Siswa. Sistem dirancang dengan prinsip Scalable System Design untuk mendukung pertumbuhan data dan pengguna sekolah dari waktu ke waktu.

Sistem tidak mencakup: sistem keuangan/pembayaran SPP, pengelolaan nilai akademik, sistem presensi berbasis RFID/biometrik, dan komunikasi orang tua secara langsung (dalam versi ini).

2. Deskripsi Umum Sistem

2.1 Perspektif Produk

Web Sekolah Terintegrasi adalah sistem informasi manajemen sekolah yang berdiri sendiri (standalone) berbasis web. Sistem ini menggantikan pengelolaan data yang sebelumnya tersebar di berbagai aplikasi atau dokumen terpisah.

Arsitektur sistem menggunakan pendekatan Modular Monolith di mana seluruh modul berjalan dalam satu aplikasi Node.js Express.js dengan satu database MySQL terpusat. Setiap modul memiliki router dan view sendiri sehingga memudahkan pengembangan dan pemeliharaan independen.

Sistem di-deploy di Oracle Cloud dengan konfigurasi Nginx sebagai reverse proxy (port 443 → 3000), PM2 sebagai process manager, dan SSL/HTTPS via Certbot. Domain publik: https://web-sekolah.duckdns.org.

2.2 Fungsi Utama Produk

  • Manajemen Pengguna: Pengelolaan akun dengan 7 role berbeda dan sistem autentikasi berbasis session
  • Data Kesiswaan: Pengelolaan data siswa, kelas, wali kelas, dan import/export CSV
  • Jurnal Mengajar: Pencatatan aktivitas mengajar harian guru per kelas dan mata pelajaran
  • Bimbingan Konseling: Pencatatan kasus, konseling, pelanggaran, dan prestasi siswa
  • Absensi Siswa: Input kehadiran harian dan rekap persentase kehadiran per siswa
  • Dashboard: Visualisasi statistik real-time dengan grafik Chart.js
  • Audit Trail: Pencatatan seluruh aktivitas pengguna di tabel activity_logs

2.4 Batasan Sistem

  • Sistem memerlukan koneksi internet untuk akses dari luar jaringan sekolah
  • Browser yang didukung: Chrome v90+, Firefox v88+, Edge v90+, Safari v14+
  • Spesifikasi minimum server: 1 vCPU, 1 GB RAM, 20 GB storage
  • Database MySQL versi 5.7 atau lebih baru
  • Node.js versi 16 atau lebih baru
  • Sistem belum mendukung multiple bahasa (hanya Bahasa Indonesia)
  • Tidak tersedia aplikasi mobile native (hanya responsive web)

2.5 Asumsi dan Ketergantungan

  • Diasumsikan setiap pengguna memiliki akun yang telah dibuat oleh Admin sebelum dapat login
  • Diasumsikan data kelas, mata pelajaran, guru, dan tahun ajaran telah diinput Admin sebelum modul lain digunakan
  • Sistem bergantung pada ketersediaan server Oracle Cloud dan koneksi internet stabil
  • Setiap guru yang ingin menggunakan modul Jurnal dan BK harus terdaftar sebagai teacher di tabel teachers oleh Admin
  • Setiap siswa yang ingin menggunakan Dashboard Siswa harus memiliki user_id yang terhubung ke akun di tabel users

5. Antarmuka Eksternal

5.1 Antarmuka Pengguna

Antarmuka pengguna berbasis web dengan tema Apple-inspired monochrome. Prinsip desain:

  • Warna: Background #f5f5f7, teks utama #1d1d1f, aksen biru #0071e3
  • Komponen: Card Bootstrap 5 dengan border-radius 12px, shadow halus, header navbar fixed-top
  • Navigasi: Sidebar/navbar horizontal; menu tampil berdasarkan role pengguna
  • Flash Messages: Alert Bootstrap 5 yang otomatis tertutup setelah 5 detik
  • Tabel: table-hover dengan pagination 10 data/halaman dan search bar
  • Modal: Bootstrap modal untuk form tambah data (BK, Users)
  • Grafik: Chart.js bar, line, doughnut, pie chart di Dashboard

Halaman login menggunakan layout guest (tanpa navbar). Semua halaman lain menggunakan layout main dengan navbar dan container-fluid.

BUG-001 — Guru BK Tidak Dapat Menyimpan Kasus / Konseling / Pelanggaran
TitleGuru BK gagal menyimpan data BK — teacher_id bernilai undefined karena akun tidak memiliki record di tabel teachers
Cara DitemukanDilaporkan langsung oleh pengguna: "masih error modul nya, gabisa tambahkan case". Dikonfirmasi melalui code review pada src/modules/bk/routes.js
DeskripsiKetika akun guru_bk yang dibuat manual oleh Admin mencoba menyimpan kasus BK, konseling, atau pelanggaran, sistem menampilkan flash error dan data tidak tersimpan. Penyebab: route mencari record di tabel teachers berdasarkan user_id dari session. Akun yang dibuat manual tidak otomatis memiliki record di tabel teachers, sehingga teacherId = undefined. INSERT ke bk_cases (kolom teacher_id NOT NULL) pun gagal.
PlatformOracle Cloud Production Server; https://web-sekolah.duckdns.org; login sebagai guru_bk
Step to Reproduce 1. Login sebagai Admin → buat akun baru role guru_bk via /users
2. Logout → login dengan akun guru_bk baru
3. Buka /bk → klik "+ Kasus" → isi form → klik Simpan
Hasil: flash error "Gagal mencatat kasus"; data tidak tersimpan
Expected ResultFlash success "Kasus BK berhasil dicatat"; data tersimpan di tabel bk_cases
Actual Result (sebelum fix)Flash error "Gagal mencatat kasus"; console server: Column 'teacher_id' cannot be null
Priority / SeverityHigh / Critical
Root Cause File: src/modules/bk/routes.js
const teacherId = teacher.length > 0 ? teacher[0].id : req.body.teacher_id;
Jika guru_bk tidak punya teacher record dan form tidak mengirim teacher_id → teacherId = undefined
Fix yang DiterapkanTambah fungsi getOrCreateTeacherId(userId, fullName) yang otomatis membuat teacher record dengan NIP BK-AUTO-{userId} jika belum ada
Commit Fix9281212 — "fix: BK - auto-create teacher record jika guru_bk belum terdaftar"
Verified (Retesting)Dikonfirmasi langsung oleh pengguna: fitur berjalan setelah fix. Test case UC05-TC01: PASS
[Screenshot: flash success setelah fix — tambah kasus BK berhasil]

BUG-003 — Record Absensi Hilang Jika User yang Menginput Dihapus
TitleQuery absensi menggunakan INNER JOIN pada kolom created_by — record absensi tidak muncul jika user yang menginput dihapus dari sistem
Cara DitemukanCode review pada src/modules/absensi/routes.js baris 30
DeskripsiQuery daftar absensi menggunakan JOIN users u ON a.created_by = u.id. Jika user yang menginput absensi (kolom created_by) dihapus dari tabel users, maka semua record absensi yang diinput oleh user tersebut tidak akan muncul di daftar absensi karena INNER JOIN tidak mengembalikan row dengan join yang gagal.
Platformhttps://web-sekolah.duckdns.org; terdeteksi dari analisis kode
Step to Reproduce 1. Login sebagai guru1 → input absensi kelas X-A tanggal tertentu
2. Login sebagai admin → hapus akun guru1
3. Buka /absensi → filter tanggal tersebut
Hasil: Record absensi yang diinput guru1 tidak muncul
Expected ResultSemua record absensi tetap muncul meski user yang menginput sudah dihapus. Kolom "Diinput oleh" boleh kosong/null
Actual Result (sebelum fix)Record absensi hilang dari daftar karena INNER JOIN gagal menemukan user yang sudah dihapus
Priority / SeverityHigh / Major
Root CauseFile: src/modules/absensi/routes.js
JOIN users u ON a.created_by = u.id seharusnya LEFT JOIN
Fix yang DiterapkanUbah JOIN menjadi LEFT JOIN users u ON a.created_by = u.id
Commit Fix8d05453 — "fix: perbaikan 12 bug di semua modul"
Verified (Retesting)Test case UC06-TC01: PASS

BUG-007 — Dropdown Search Suggestion Tidak Dapat Diklik
TitleSaat pengguna mengklik item pada dropdown suggestion, dropdown langsung tertutup sebelum klik diproses karena event blur pada input field terpicu lebih dulu
Cara DitemukanPengujian langsung — klik dropdown tidak berfungsi di semua modul (Users, Kesiswaan, Jurnal, BK, Absensi)
DeskripsiKetika pengguna mengetik di search bar dan dropdown suggestion muncul, mengklik item di dropdown tidak berhasil. Dropdown tertutup saat mouse ditekan karena blur event pada input terpicu sebelum click event pada dropdown item. Akibatnya search suggestion tidak berguna sama sekali.
Platformhttps://web-sekolah.duckdns.org; semua halaman dengan search bar; Chrome v126
Step to Reproduce 1. Login sebagai admin → buka /kesiswaan
2. Ketik "Ahmad" di search bar — dropdown muncul
3. Gerakkan mouse ke dropdown → klik salah satu item
Hasil (sebelum fix): Dropdown hilang sebelum item diklik; tidak ada aksi yang terjadi
Expected ResultKlik item dropdown → input terisi nama siswa → form disubmit / halaman terfilter
Actual Result (sebelum fix)Dropdown hilang saat mouse button ditekan (mousedown) sebelum click event diproses
Priority / SeverityHigh / Major
Root CauseFile: src/public/js/search-suggest.js — tidak ada mousedown: e.preventDefault() pada dropdown untuk mencegah blur event pada input. Urutan event: mousedown (dropdown) → blur (input) → dropdown hilang → click tidak diproses
Fix yang DiterapkanTambah dropdown.addEventListener('mousedown', e => e.preventDefault()) untuk mencegah blur event saat klik dropdown. Juga pindahkan event listener ke event delegation pada dropdown container
Commit Fixc13cd47 — "fix: search suggestion klik & transparansi"
9fcd3ab — "fix: search suggest klik & hover effect"
Verified (Retesting)Klik dropdown bekerja di semua modul setelah fix. Test case UC04-TC05: PASS
[Screenshot: dropdown search suggestion terbuka + klik item berhasil — tabel terfilter]
